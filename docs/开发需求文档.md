# 个人股票分析系统 — 开发需求文档

> 版本: v1.0 | 更新日期: 2026-02-12

## 一、产品概述

基于 Streamlit + Claude API + SQLite 的个人 A 股投资分析系统。
核心理念：**数据采集 → AI清洗结构化 → 多维度深度研究 → 热点发现 → 知识图谱沉淀 → 持仓跟踪**，形成完整的投资研究闭环。

### 技术栈

| 层 | 技术 |
|---|---|
| 前端 | Streamlit (Python Web UI) |
| 数据库 | SQLite (WAL模式) |
| AI引擎 | Claude API (claude-sonnet-4-20250514) |
| 行情数据 | AKShare (免费A股数据) |
| 分析框架 | Claude Code Skills (多步骤Skill驱动分析) |
| 可视化 | Plotly (K线/图表), PyVis (知识图谱) |

---

## 三、数据库设计 (7层23张表)

### 第1层：原始数据
- `data_sources` — 数据源注册表（名称、类型、速率限制）
- `raw_items` — 原始采集条目（标题、内容、URL、处理状态）

### 第2层：清洗数据
- `cleaned_items` — 结构化清洗结果（事件类型、情感、重要性、摘要、标签）
- `item_companies` — 条目关联公司（股票代码、关联度、影响方向）
- `item_industries` — 条目关联行业
- `research_reports` — 券商研报（评级、目标价、分析师）

### 第3层：行情数据
- `stock_info` — 股票基础信息（代码、名称、行业、市值）
- `stock_daily` — 日线行情（OHLCV、换手率、涨跌幅）
- `capital_flow` — 个股资金流向（主力/超大单/大单/中单/小单）
- `industry_capital_flow` — 行业资金流向
- `northbound_flow` — 北向资金净流入
- `macro_indicators` — 宏观经济指标
- `financial_reports` — 财务报告（营收、净利、YoY、EPS、ROE）

### 第4层：知识图谱
- `kg_entities` — 实体（宏观因子/行业/公司/主题/指标）
- `kg_relationships` — 关系（强度、方向、置信度）
- `kg_update_log` — 变更日志
- `kg_verification` — 关系验证记录

### 第5层：研究与机会
- `deep_research` — 深度研究报告（6维评分、推荐等级）
- `investment_opportunities` — 投资机会（评级A/B/C、来源）
- `pipeline_runs` — 流水线执行日志
- `api_usage` — API用量追踪

### 第6层：持仓跟踪
- `watchlist` — 自选股（interested/holding）
- `watchlist_tags` — 标签自选
- `holding_positions` — 持仓明细（成本、数量、盈亏）
- `holding_research_log` — 持仓自动研究日志
- `portfolio_reviews` — 组合回顾报告

### 第7层：热点动向
- `dashboard_tag_frequency` — 8类榜单标签频次
- `tag_groups` — 标签组（组名、标签、投资逻辑、频次）
- `tag_group_research` — 标签组深度研究

## 四、功能模块详细需求

### 模块1：数据采集 (ingestion/)

#### 1.1 AKShare行情采集 (`akshare_source.py`)

| 功能 | 函数 | 说明 |
|------|------|------|
| A股基础信息 | `fetch_stock_info()` | 拉取全市场股票代码+名称，写入stock_info |
| 个股详情 | `fetch_stock_detail(code)` | 行业、市值、总股本、流通股、上市日期 |
| 日线行情 | `fetch_stock_daily(code, start, end)` | OHLCV+换手率+振幅+涨跌幅，前复权 |
| 资金流向 | `fetch_capital_flow(code)` | 主力/超大单/大单/中单/小单净流入 |
| 财务摘要 | `fetch_financial_data(code)` | 同花顺财务摘要（营收、净利、YoY、EPS、ROE） |
| 行业资金流 | `fetch_industry_capital_flow()` | 行业板块资金流向排名 |
| 北向资金 | `fetch_northbound_flow()` | 北向资金每日净流入 |
| 批量拉取 | `fetch_all_daily_data(codes)` | 批量拉取watchlist股票的日线+资金流 |

### 模块2：AI清洗 (cleaning/)

#### 2.1 Claude结构化清洗 (`claude_processor.py`)

核心函数 `clean_single_item(raw_item_id)`，处理流程：

1. 从raw_items取原始数据
2. 内容校验（<10字跳过，>3000字截断）
3. 调Claude API做结构化提取
4. 保存到cleaned_items + item_companies + item_industries + research_reports
5. 更新raw_items状态为cleaned

清洗输出字段：

| 字段 | 说明 |
|------|------|
| event_type | macro_policy / industry_news / company_event / earnings / research_report |
| sentiment | positive / negative / neutral（投资视角） |
| importance | 1-5分（5=重大政策/颠覆性事件） |
| summary | 结构化摘要 |
| key_points | 关键要点列表 |
| tags | 标签（行业标签+主题标签+宏观标签） |
| companies | 关联公司（代码、名称、关联度primary/secondary/mentioned） |
| industries | 关联行业 |
| impact_analysis | 影响分析 |
| time_horizon | short / medium / long |
| confidence | 0-1置信度 |

增强清洗 `clean_with_event_analysis(raw_item_id)`：对重要性≥4的条目追加深度事件分析。

#### 2.2 批量清洗 (`batch_cleaner.py`)

- 支持进度回调
- 并发控制
- 重试逻辑
- 流水线执行追踪（pipeline_runs表）

### 模块4：深度研究引擎 (research/)

#### 4.1 数据充分性检查 (`data_readiness.py`)

深度研究前的前置检查模块，确保数据充分再跑分析：

| 维度 | 数据表 | 最低阈值 | 自动补数据方式 |
|------|--------|----------|---------------|
| 基础信息 | stock_info | 必须有行业+市值 | `fetch_stock_info()` + `fetch_stock_detail()` |
| 日线行情 | stock_daily | ≥60个交易日 | `fetch_stock_daily()` (近180天) |
| 资金流向 | capital_flow | ≥20个交易日 | `fetch_capital_flow()` |
| 财务数据 | financial_reports | ≥2个报告期 | `fetch_financial_data()` |
| 相关新闻 | cleaned_items | ≥3条 | `_fetch_stock_news()` + `clean_single_item()` |
| 券商研报 | research_reports | ≥0（不阻塞） | 无自动补充 |

核心流程 `ensure_stock_data_ready(code, max_rounds=2)`：
1. 第1轮：check → 发现缺失 → 自动补数据（含新闻采集+清洗）
2. 第2轮：re-check → 仍不足则记warning但继续
3. 返回readiness报告（ready/confidence/dimensions/missing）

置信度计算：达标维度占比 + 研报≥3篇额外+5%

#### 4.2 个股深度研究 (`deep_researcher.py`)

5步Skill驱动分析 `deep_research_stock(code)`：

| 步骤 | Skill | 说明 |
|------|-------|------|
| 0 | — | 数据充分性检查 + 自动补数据 |
| 1 | financial-model-construction | 财务建模分析（业务拆解） |
| 2 | company-valuation | 估值分析（基于Step1结果） |
| 3 | stock-chart-analysis | 技术面分析 |
| 4 | dynamic-stock-chart-predict | 买卖点预测（基于Step3结果） |
| 5 | stock-event-analysis | 事件与情绪分析 |
| 综合 | STOCK_SYNTHESIS_PROMPT | 6维评分 + 投资结论 |

6维评分体系：
- financial_score (0-100) — 财务质量
- valuation_score (0-100) — 估值水平
- technical_score (0-100) — 技术面
- sentiment_score (0-100) — 市场情绪
- catalyst_score (0-100) — 催化剂
- risk_score (0-100) — 风险（越高=越安全）
- overall_score (0-100) — 综合加权

推荐等级：强烈推荐 / 推荐 / 中性 / 谨慎 / 回避

overall_score ≥ 70 自动生成投资机会（A/B/C评级）。

研究context中附加数据充分性信息（置信度+各维度达标状态）。

#### 4.3 行业深度研究

2步分析 `deep_research_industry(industry_name)`：
1. event-industry-impact — 行业结构与竞争分析
2. macro-stock-analysis — 宏观环境对行业的影响

输出：industry_score / growth_score / policy_score / 推荐个股

#### 4.4 宏观深度研究

2步分析 `deep_research_macro(topic)`：
1. macro-stock-analysis — 宏观经济指标分析
2. macro-insight-generation — 宏观洞察与投资启示

输出：cycle_stage / market_outlook / confidence / 行业配置建议

#### 4.5 统一数据访问 (`universal_db.py`)

| 函数 | 说明 |
|------|------|
| `get_stock_profile(code)` | 聚合stock_info + daily(250天) + capital(60天) + financials(8期) + news(30条) + reports(20篇) |
| `get_stock_technical_summary(code)` | 计算MA5/10/20/60、5日/20日涨跌、60日高低点、均量/均换手 |
| `get_peer_comparison(code)` | 同行业市值Top10对比 |
| `get_industry_data(name)` | 行业资金流+成分股+行业新闻 |
| `get_macro_data()` | 宏观指标+北向资金+宏观新闻 |

### 模块5：热点研究 (hotspot/)

#### 5.1 标签频次分析与标签组推荐 (`tag_recommender.py`)

标签组生成流程（共现聚类 + Claude投资逻辑验证）：

1. 从 `dashboard_tag_frequency` 取近N天Top40高频标签
2. 构建共现矩阵：同一天同一榜单出现的标签视为关联
3. 贪心聚类：从最高频标签对开始扩展成2-5个标签的候选组
4. 多生成3倍候选组（top_n × 3）
5. 调Claude验证每个候选组的投资逻辑：
   - 判断标签间是否有因果/产业链/政策传导/资金轮动关系
   - 过滤无逻辑关联的组合（如"降准+消费电子+白酒"）
   - 为有效组合生成投资主线名称和一句话逻辑
   - 可裁剪组内不相关标签
6. 返回验证通过的标签组，按频次排序

附加功能：
- `get_group_related_news(tags, days, limit)` — 获取标签组关联的清洗后新闻摘要
- `save_tag_group()` / `get_saved_groups()` / `delete_tag_group()` — 标签组持久化

#### 5.2 标签组快速分析 (`tag_group_analyzer.py`)

`analyze_tag_group(tags, days)` 三维分析：
1. 成组逻辑 — Claude分析标签间内在联系、投资主线、持续性
2. 关联新闻排序 — 按（匹配标签数 × 重要性）排序
3. 板块热度 — 行业资金流向时序 + 个股资金流向汇总

#### 5.3 标签组深度研究 (`tag_group_research.py`)

`research_tag_group(group_id)` 生成6维研究报告：
- 宏观分析报告
- 行业分析报告
- 关联新闻汇总
- 板块热度数据
- Top10推荐个股
